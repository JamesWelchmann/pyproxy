API Reference¶

pyproxy is the client library for the PyProxy Server

class pyproxy.Future(id, inner_fut)¶

PyProxy Client Future

is_done()¶

True if done, False otherwise this function is exception safe

wait(timeout=None)¶

wait for a future to complete this will also raise any exception the future wishes to raise

We can call wait as many times as we like it will reliably produce the same behaviour once done

wait_no_except(timeout=None)¶

wait_no_except is the same as wait except that it will return an exception, not raise it

exception pyproxy.PyProxyClosedSessionError¶

PyPRoxyClosedSessionError is raised when we attempt to perform actions on a PyProxySession which is already disconnected. The most likely reason is you called RemoteProcess.eval on an already closed session.

exception pyproxy.PyProxyError¶

PyProxyError is the base exception of PyProxy Client

exception pyproxy.PyProxyIOError¶

PyProxyIOError is raised when I/O fails on either mainstream or outputstream

exception pyproxy.PyProxyProtocolError¶

PyProxyProtocolError is raised when client/server protocol sent invalid bytes/messages.It is used by both mainstream and outputstream

class pyproxy.PyProxySession(addr='localhost:9000')¶

PyProxySession is a class for establishing the mainstream with the PyProxy Server.

connect()¶

connect will spawn a background thread we return a RemoteProcess object

property session_id¶

session_id, as sent in the server-hello message

class pyproxy.RemoteProcess(client)¶

RemoteProcess is our object wrapping a single “remote python process”, we can send code to execute and interact with stdin/stdout and stderr.

eval(code, locs=None, globs=None)¶

eval will execute a code object on the remote process code may be a str or a code object

if code_object is a string then we send to the server as a String if code_object is a code_object then we attempt to pickle

output(fut=None)¶

output retrieves stdout and stderr line from the remote process

stdin(lines)¶

stdin will send lines to stdin on the remote process. Currently we raise NotImplementedError
